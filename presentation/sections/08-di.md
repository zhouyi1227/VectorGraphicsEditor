# 继承体系

<div class="deck-rule"></div>
<div class="deck-lead">
  项目里的继承分两类：一类承接 Qt 已有的窗口与图形框架，一类把创建流程抽象成运行时可切换的策略。
</div>

::left::

```mermaid
flowchart TB
  QGO[QGraphicsObject] --> SI[ShapeItem]
  QGO --> OI[SelectionOverlay]
  QGV[QGraphicsView] --> CV[CanvasView]
  QMW[QMainWindow] --> MW[MainWindow]
  QW[QWidget] --> PP[PropertyPanel]
  QD[QDialog] --> TD[TutorialDialog]
```

<div class="diagram-caption">Qt 继承链：界面对象挂在最合适的宿主类型上。</div>

::right::

```mermaid
flowchart TB
  I[ICreationStrategy] --> D[DragCreationStrategy]
  I --> P[PathCreationStrategy]
  D --> S[Line / Circle / Ellipse / Rectangle]
  P --> T[Polyline / Polygon]
```

<div class="diagram-caption">业务继承链：同一组鼠标事件切换成不同的创建行为。</div>

<div v-click class="soft-copy mt-4 leading-6">
  左链复用 Qt 信号槽和事件分发；右链让 <code>CanvasView</code> 不为每种图形写 if / else。
</div>
