# 文件格式与 I/O

<div class="deck-rule"></div>

<div class="deck-lead">
  文件格式采用自定义的 <code>.vgjson</code>。它牺牲了一些紧凑性，换来了更清晰的字段语义、更直接的调试体验，以及更容易验证的输入输出契约。
</div>

```json
{
  "version": 2,
  "canvas": { "width": 1200, "height": 800 },
  "shapes": [ ... ]
}
```

<div class="diagram-caption">单个 shape 再细分为 <code>id / type / geometry / style / transform / zValue</code>，由 <code>shapeDataToJson()</code> 与 <code>shapeDataFromJson()</code> 统一编解码。</div>

<div class="split-even mt-5">
  <div class="rail-list">
    <div v-click class="rail-item">
      <div class="rail-index">SAVE</div>
      <div class="rail-title">保存路径</div>
      <div class="rail-copy"><code>FileManager::saveToFile()</code> 先检查 <code>transform</code> 是否受支持，再组装 <code>version + canvas + shapes</code> 根对象并写入缩进 JSON。</div>
    </div>
    <div v-click class="rail-item">
      <div class="rail-index">LOAD</div>
      <div class="rail-title">加载路径</div>
      <div class="rail-copy"><code>loadFromFile()</code> 会验证根对象、文档版本、图形反序列化结果、变换合法性和非空 ID 唯一性。</div>
    </div>
  </div>
  <div class="deck-stage">
    <div class="eyebrow">契约选择</div>
    <div class="thin-divider"></div>
    <div v-click class="statement-band">
      当前文件格式只接受 <strong>version 2</strong>，并明确拒绝旋转 transform。
      这样做换来的是更稳定、可测试的缩放与保存行为。
    </div>
  </div>
</div>
