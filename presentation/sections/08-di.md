# 继承体系

<div class="deck-rule"></div>
<div class="deck-lead">
  项目里的继承关系分成两类：一类承接 Qt 已有的窗口与图形框架，一类把创建流程抽象成可切换的运行时策略。
</div>

::left::

```mermaid
flowchart TB
  QGO[QGraphicsObject] --> SI[ShapeItem]
  QGO --> OI[SelectionOverlay]
  QGV[QGraphicsView] --> CV[CanvasView]
  QMW[QMainWindow] --> MW[MainWindow]
  QW[QWidget] --> PP[PropertyPanel]
  QD[QDialog] --> TD[TutorialDialog]
```

<div class="diagram-caption">Qt 继承链回答“每个界面对象应该挂在哪个成熟宿主类型上”。</div>

::right::

```mermaid
flowchart TB
  I[ICreationStrategy] --> D[DragCreationStrategy]
  I --> P[PathCreationStrategy]
  D --> S[Line / Circle / Ellipse / Rectangle]
  P --> T[Polyline / Polygon]
```

<div class="diagram-caption">业务继承链回答“同样的鼠标输入为何能切换成不同图形创建行为”。</div>

<div v-click class="soft-copy mt-4 text-base leading-7">
  真正重要的不是“画出多少继承箭头”，而是让 Qt 框架复用和业务多态扩展各自站在合适的位置上。
</div>
