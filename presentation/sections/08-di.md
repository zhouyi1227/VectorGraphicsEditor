# 依赖注入 · CreationContext

<div style="height:2px;width:32px;background:#2d7ff9;margin:0.5rem 0 1.5rem 0;"></div>

<div class="dim" style="font-size:0.85rem;margin-bottom:1rem;">策略对象不依赖 CanvasView，只依赖回调接口</div>

<div class="grid" style="display:grid;grid-template-columns:1fr 1fr;gap:1.5rem;">

<div>

```cpp
struct CreationContext {
    std::function<qreal()>
        nextZValue;
    std::function<ShapeStyle(ShapeType)>
        currentStyleFor;
    std::function<void(
        const ShapeData&, bool)>
        addShape;
    std::function<QString()>
        generateId;
    std::function<
        std::optional<ShapeType>()>
        currentShapeType;
    QGraphicsScene* scene;
};
```

</div>

<div style="border:1px solid #e8e8e8;border-radius:6px;padding:1rem;">

<div style="font-weight:600;font-size:0.9rem;margin-bottom:0.5rem;">注入方式</div>

```cpp
CreationContext ctx;
ctx.nextZValue = [this] {
    return ++m_nextZValue;
};
ctx.addShape = [this](
    auto& data, bool preview) {
    addShapeToScene(data, preview);
};

auto strategy =
    make_unique<
      DragCreationStrategy>(ctx);
```

CreationContext包含6个std::function回调和一个scene指针，通过构造函数注入到策略对象中。策略在创建过程中需要查询z值、获取样式、生成ID、添加图形时，都通过调用这些回调完成，完全不感知CanvasView的存在。好处：策略和视图完全解耦，可以独立测试；测试时可注入mock回调验证策略行为；新增策略类型不需要修改CanvasView的事件处理代码。
